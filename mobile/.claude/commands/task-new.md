## Guidelines

### 1. **Determine Task Type**

- **Bug** → `docs/tasks/bugs/bug-{short-name}.md`
- **Enhancement/Feature** → `docs/tasks/enhancements/enhancement-{short-name}.md`

### 2. **Follow Format from RULES.md**

Read `docs/tasks/RULES.md` for the exact format template. The task file must follow the standard header format defined there.

### 3. **Task Analysis**

Before creating the file:
1. Understand the requirements
2. Identify affected files/features
3. Estimate complexity (Small/Medium/Large)
4. List implementation steps

### 4. **Implementation Steps**

Break down into clear, sequential steps:
- [ ] Step 1: Investigation/setup
- [ ] Step 2: Core changes
- [ ] Step 3: Testing
- [ ] Step 4: Cleanup

## What to Do

1. Read `docs/tasks/RULES.md` for format
2. Create the task file in the appropriate directory
3. Fill in all required fields
4. List implementation steps as checkboxes
