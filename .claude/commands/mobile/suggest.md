## What to Analyze

1. **Read** the relevant code in the codebase
2. **Evaluate** against these areas:
   - UX improvements (user flow, feedback, accessibility)
   - Code quality (readability, maintainability, patterns)
   - Performance (rendering, data fetching, memory)
   - Reliability (error handling, edge cases, offline)
   - Mobile best practices (platform conventions, gestures, animations)

3. **Prioritize** suggestions by impact:
   - **High Impact** — Directly improves user experience or fixes real problems
   - **Medium Impact** — Improves code quality or developer experience
   - **Low Impact** — Nice-to-have, cosmetic, or minor
