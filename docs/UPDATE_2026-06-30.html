Engineering Update

Admin AI Tooling, Reliability Fixes & a Smarter Dashboard

New AI features, a recovered customer-chat bug, and platform hardening — all shipped to production.

Date:  June 30, 2026 Status:  ✓ Delivered & Pushed to main Commits:  4 shipped

Summary

Today we delivered two new AI capabilities for the admin team, fixed the root cause of a recurring dashboard crash, recovered a customer-facing AI chat that had silently stopped working, made the support assistant noticeably more personal, and added a fast keyboard-driven way to move around the admin dashboard. Everything is verified and pushed to production.

1 Sessions crash — fixed at the root, with a permanent guard

The admin Sessions page was crashing because the database contained duplicate administrator records (the same wallet inserted multiple times over several months).

  • Ran a read-only review first, confirmed no foreign-key references, then removed 9 duplicate rows (keeping the oldest, real-name record per wallet). 0 duplicates remain.
  • Shipped Migration 189 — it de-duplicates admin records and adds a unique index so duplicates can never be created again. Idempotent and self-healing: it runs automatically on deploy, so production is cleaned up and protected with no manual work.

Verified the new guard rejects duplicate inserts:

ERROR:  duplicate key value violates unique constraint "uq_admins_wallet_lower"

2 Recovered the customer AI chat BUG FIX

While cleaning up types we discovered the AI repair-assistant chat was effectively non-functional: the service returned the inner data object while the chat screen expected the full response wrapper, so every internal check silently failed and messages never processed.

Impact: the fix restores customer AI chat (start, send, image upload, history) — a customer-facing feature that had been quietly broken.

3 AI Bug Inspection NEW

Bug reports now have an “AI Inspect” button. With one click, AI reads the report and explains the most likely cause of the problem — so the team can act faster.

  • Probable cause stated up front, plus a short root-cause analysis.
  • Affected area (e.g. billing/webhooks, wallet, booking, UI), severity, and a confidence level.
  • Suggested investigation/fix steps, with a one-click re-run.

4 More personal support replies

The support assistant now reads the full conversation and the shop’s name first, and is instructed to respond to what the shop actually said — referencing their own words — instead of a generic acknowledgement. Replies are warmer and specific to each ticket.

5 Smart Command Bar NEW

A keyboard-driven command palette across the entire admin dashboard — press ⌘K (or Ctrl+K), or click the floating button.

  • Instant navigation — fuzzy-search all 21 admin sections and jump there with the keyboard.
  • Ask AI — type a question (e.g. “how many shops are pending?”) and get an answer inline, powered by the Platform Copilot, without leaving the page.
Why it matters: faster day-to-day operations for the admin team — navigate and ask questions in seconds instead of clicking through menus.

6 Production audit & documentation

  • Verified production runs in database-only mode, backend healthy, all AI endpoints deployed, and the blockchain-UI hiding fully wired.
  • Identified that the AI features need their API keys configured in the deploy environment to produce real output (they run in a safe fallback mode until then).
  • Wrote an operations runbook with the exact steps to enable the AI keys and verify the deploy, plus a client-facing update document.
