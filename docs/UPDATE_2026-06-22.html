3 Shop "Subscription Expired" Fix & Tier Data Cleanup Resolved

Investigated a shop showing a "Subscription Expired" banner despite being a qualified (RCG-holding) shop, and found a systemic data bug behind it.

Diagnosis

  • The shop was actually healthy server-side (RCG-qualified) — the banner was stale browser cache. A refresh clears it.

Root cause & fix

  • Found a seed-data bug: every shop's tier was labeled "STANDARD" (wrong casing) regardless of its actual token balance — breaking RCN pricing, tier benefits, and analytics.
  • Backfilled 59 shops: recomputed each shop's tier from its real balance (none / standard / premium / elite). No shop's access was changed — only the (incorrect) labels.
  • Corrected the specific shop to its real Premium tier.

Database correction (staging). Final distribution: 57 none, 2 standard, 1 premium.
