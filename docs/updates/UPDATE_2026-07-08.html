Engineering Update

Stabilizing the New Admin Tools & Sidebar Cleanup

Bug fixes, matching the data to the current (blockchain-off) reality, and a categorized sidebar.

Date:  July 8, 2026 Status:  ✓ Delivered & Pushed to main

Summary

A polish-and-fix day. We hardened the admin tools shipped over the last two days (several returned bad data or errored on real data), corrected figures that didn't match the current blockchain-off reality, parked the affiliate feature, and reorganized the now-crowded admin sidebar into clean categories.

1 Fixes FIX

  • Referrals tab (was erroring / 500). It relied on a database view that doesn't exist in this environment. Rewired to compute the top-referrer leaderboard directly from the referrals table — now loads with real data.
  • Revenue "by tier" garbled numbers. The database returns money values as text, so the code was concatenating them (e.g. $025.501.00…) and showing NaN% discount. Fixed the arithmetic; the tier breakdown now reconciles correctly.
  • Audit Log crash. A log entry with no action value crashed the whole table. Made it null-safe so any entry renders gracefully.
  • Webhook Monitor false “degraded”. The health check depended on a database function missing from this environment, which errored and falsely reported degraded. Now computes health directly from the logs (no recent failures = healthy).
  • Marketplace tab color mismatch. It used light cards on the dark dashboard; re-themed to match the rest of the admin tabs.

2 Matched to Current Reality ACCURACY

  • Revenue — hid Stakers / DAO / projections. There are no stakers or DAO yet (those are future blockchain-governance features), so showing an 80/10/10 revenue split was misleading. Gated behind the blockchain flag, so Revenue now shows only real platform revenue and shop-tier discounts. The split and projections return automatically when blockchain is re-enabled at launch.
  • Affiliate Groups parked. Confirmed the feature is still fully wired (backend + shop pages), just hidden from the shop menu. Hid the admin menu link too, for consistency. (The 122 groups in the data turned out to be ~87% QA/test entries — only two hold real balances.)

3 Sidebar Cleanup HOUSEKEEPING

The admin sidebar had grown to two dozen items in one long list. We grouped everything under labeled categories — Users & Shops · Finance & Tokens · Growth & Analytics · AI Tools · Trust & Safety · Communication · System — with Overview on top and Settings pinned at the bottom. Navigation is unchanged; it's now much easier to scan.

Verification

Backend type-checking clean (0 errors); the frontend gained 0 new type errors across every fix; each change was committed and pushed independently, rebased onto the latest main.

Where Things Stand

The eight admin tools built over the prior two days are now stable, accurate, and consistent with the current database-only (blockchain-off) mode. The admin surface is organized and dependable.

The standing unlock: enabling the two AI API keys in production makes the full AI suite go live; re-enabling the blockchain flag at launch brings back the token-governance features (stakers/DAO split, staking, affiliate tokens) automatically.
