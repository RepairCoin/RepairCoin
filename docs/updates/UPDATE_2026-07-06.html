Engineering Update

Surfacing Hidden Admin Power — 4 New Tools

A gap analysis found major admin features built in the backend but never exposed. We shipped four of them.

Date:  July 6, 2026 Status:  ✓ Delivered & Pushed to main Commits:  5 shipped

Summary

We ran a structured gap analysis of the admin dashboard and discovered a large amount of functionality that existed in the backend but was never surfaced to admins. Today we turned four of those into real, shipped admin tools — an Audit Log, a Revenue Analytics dashboard, a Webhook Monitor, and an RCG Management tab — and cleaned up ~1,650 lines of dead code along the way. Every feature reuses existing backend endpoints, so risk was low and delivery fast (3 of the 4 needed zero backend changes).

1 Audit Log NEW

Every admin action — suspensions, token mints, RCN sales, role changes, admin creation/deletion, contract pauses, emergency stops — was already being recorded in the database, but there was no way to view it. There's now a full Audit Log tab:

  • Table of When · Admin · Action · Target · Details, newest first.
  • Colour-coded action badges by risk (red for suspend/delete/emergency, green for activate/approve, yellow for mints/sales, violet for admin/role changes).
  • Filters by admin wallet and target type, with pagination.
Why it matters: real accountability now that there are multiple admins and a create-admin flow — every privileged action is visible and searchable.

2 Revenue Analytics NEW

A complete revenue reporting suite existed in the backend but had no interface. The new Revenue tab shows:

  • This-week & custom-range revenue with the platform's 80/10/10 split (Operations / Stakers / DAO).
  • Revenue by shop tier (last 30 days) — including how much revenue is given away as RCG tier discounts (actual vs. standard pricing).
  • A staker-revenue projection calculator — enter monthly volume + tier to project monthly/annual staker revenue and APR.

3 Webhook Monitor NEW

FixFlow & Stripe webhook failures were effectively invisible — a failed reward mint or missed subscription event would just be lost. The new Webhook Monitor makes them visible and recoverable:

  • Health banner (healthy / degraded) with failed-24h and pending counts.
  • Log table — event, source, status (colour-coded + HTTP code), retry count, and the actual error message.
  • Filters by status and source, plus one-click Retry to reprocess a failed webhook.
Note: these endpoints were added behind proper admin authentication, and the console shows only real data (the backend's placeholder “stats” were deliberately not used).

4 RCG Management NEW

Governance-token oversight had no UI. The new RCG tab surfaces supply (total / circulating / held-by-shops), shop tier distribution, top RCG holders, and an OTC sale recorder that validates the shop and prices the Standard/Premium/Elite packages. We skipped two related endpoints (OTC request tracking, commitment enrolments) because they are backend stubs with no data yet.

5 Codebase Cleanup CLEANUP

The gap analysis also found orphaned UI — components built but never rendered. We removed six dead/superseded components (~1,650 lines): three old shop tabs replaced by the unified Shop Management screen, a superseded activity section, an empty stub tab, and a dead parallel chart library. This also cleared 8 lingering type errors.

Where Things Stand

Four significant admin capabilities that were previously invisible are now live and usable. The admin surface is materially more powerful — full accountability, revenue visibility, webhook recoverability, and governance-token oversight — with a cleaner codebase behind it.

Remaining opportunities: a notification/announcement broadcast composer, an affiliate-groups admin view, and a referral-analytics dashboard. Plus the standing unlock — enabling the AI API keys in production to make the full AI suite live.
