Engineering Update

Smarter ⌘K, a Full Tab Audit & the Fixes It Found

An intent-aware command bar, a complete review of every admin tab, and same-day fixes for everything it surfaced.

Date:  July 1, 2026 Status:  ✓ Delivered & Pushed to main Commits:  7 shipped

Summary

A heavy, high-impact day focused on the admin experience. We made the ⌘K Command Bar genuinely intelligent (it now understands intent and answers with live data), ran a complete audit of all 21 admin tabs, and then fixed everything the audit surfaced — including three fully broken features, several actions that silently did nothing, and misleading numbers on the main dashboard. Every change was type-checked, verified, and shipped in small, reviewable commits.

1 The Smart Command Bar (⌘K) Got Much Smarter NEW

Earlier we shipped a ⌘K command palette for quick navigation. Today we upgraded its “Ask AI” from a plain text box into an intent-aware assistant.

  • Understands what you mean, not just what you type. A new backend brain first classifies your query as either “take me somewhere” or “answer a question”:
    • “approve a shop” → jumps straight to Pending Shops
    • “check for fraud” → jumps to Fraud Detection
    • “how many shops are pending?” → answers with real, live platform data
  • Suggests next steps — every answer comes with clickable section chips.
  • Fails gracefully — if AI isn't configured, instant keyword navigation still works.
  • Cross-platform — the label is now platform-aware (Ctrl K on Windows/Linux, ⌘K on Mac).

2 A Complete Audit of All 21 Admin Tabs

We systematically reviewed every admin tab (~22,000 lines of code) using four parallel reviewers, checking each for mock/placeholder data, dead buttons, actions with no backend wiring, missing features, and silent failures. The large majority of tabs were complete and fully wired; the audit then pinpointed a focused set of real problems, which we fixed the same day.

3 Critical Blockers Fixed FIX

The audit uncovered three genuinely broken things (confirmed by the type-checker, but hidden in production because the build suppresses type errors):

  • Create Admin was completely broken. It referenced an undefined variable and would throw the moment you clicked “Create Admin.” Rewired to the correct authenticated call.
  • Shops → Unsuspend Requests crashed on load. The same class of bug (a leftover from an older auth approach). Fixed to use the current cookie-based authentication.
  • Dead legacy Treasury tab removed — an old, unused component full of broken references and “coming soon” placeholders (~1,000 lines).

4 Content Moderation — Flagged Reviews Are Now Actionable FIX

Previously the tab could flag inappropriate reviews but gave admins no way to act on them (the only button worked for service listings, not reviews). Admins can now permanently remove a flagged review, protected by an inline two-step confirmation. This closed the single biggest functionality gap the audit found.

5 Dashboard Overview — Honest Numbers FIX

  • “Total Revenue” was faked — calculated as tokens issued × $0.10. It now shows the real revenue figure (actual shop RCN purchases).
  • The “Redemptions” transaction filter was a dead end — it always returned empty. Removed so every filter returns real results.

6 Actions That Silently Did Nothing FIX

  • Unsuspend “Reject” was a no-op. In both the Customer and Shop detail modals, “Reject” showed a success message but never rejected anything. Both now call the real endpoint.
  • Silent load failures. Bug Reports and Disputes only logged fetch errors to the console — so a failed load looked identical to an empty list. Both now show a clear error message.
