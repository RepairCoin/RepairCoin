SUMMARY
-------
Pulled the latest main and executed the go-live checklist against the running
app — config sanity, the "blockchain hidden" launch requirement, and the trial
+ billing flow. The config wiring and shop surface held up clean, but the audit
surfaced a set of leftover blockchain leaks (admin RCG/Treasury, two customer QR
headings, and stray public-page wording) that the July 9 pass didn't reach. All
are now fixed, type-checked, and pushed. Two production billing blockers were
identified and handed off.


§0  CONFIG SANITY — VERIFIED
----------------------------
Booted the backend against the live database and confirmed the launch config
end-to-end:
  - GET /api/config returns { blockchainEnabled: false } — verified live, not
    just in source.
  - The frontend config context fails closed: it defaults to blockchain-hidden
    and stays hidden if the config call ever errors, so nothing crypto can flash
    on a slow network.
  - Wallet login is confirmed independent of the flag — Thirdweb connect renders
    unconditionally, exactly as intended (we keep login, hide everything else
    blockchain).
  - ANTHROPIC / OPENAI keys present; Stripe keys present (test locally — prod
    needs live keys, see blockers).


§1  BLOCKCHAIN HIDDEN — LEAK AUDIT & FIXES — FIXED
--------------------------------------------------
Audited every user-facing surface for wallet/blockchain/token UI that could
still render with the flag off. The shop surface was already fully gated
(Wallet & Payouts, staking, RCG, mint, crypto pay — all correctly hidden). The
following leaks were found and closed:

  - Admin (largest gap): the RCG nav link + tab still rendered, and inside
    Treasury the Stakers (10%) / DAO (10%) revenue-share cards and the RCG
    Metrics sub-tab were ungated. Now all gated behind the blockchain flag; the
    legitimate RCN-sales, pricing and analytics content in Treasury stays
    visible.
  - Customer: two QR modal headings were hardcoded "Wallet Address QR Code" —
    now use the shared label and read "RepairCoin ID QR Code" (redemption +
    settings).
  - Public / legal: dropped leftover blockchain/crypto/staking wording from the
    privacy policy, delete-account page, the sign-in "choose" page, the
    team-invite page, the tier explainer, and the public shop tier cards.

  Reversible: the admin and customer fixes are behind ENABLE_BLOCKCHAIN_MINTING
  and restore automatically when the flag flips back on; the public copy is
  neutral in both modes.


§3  TRIAL + BILLING — VERIFIED (2 BLOCKERS)
-------------------------------------------
Traced the tier/trial/checkout flow across frontend and backend:
  - Prices match front-to-back — Starter $80 / Growth $299 / Business $599,
    14-day trial constant, and the plan-comparison UI is wired into the working
    checkout. OK
  - Blocker 1 — Stripe tier price IDs: STRIPE_PRICE_STARTER / _GROWTH / _BUSINESS
    are not set; checkout silently falls back to the single legacy price, so all
    three tiers would bill the same amount. Create the three prices in Stripe and
    set them in prod env.
  - Blocker 2 — Live Stripe keys: production must use live keys + webhook secret
    (local is test mode).
